# 记忆余烬 - 音乐音效流程文档 (Audio Workflow)

## 一、音频架构总览

### 音频引擎：Web Audio API + Howler.js（后备）
- **主控模块**：`AudioManager` 全局单例（JavaScript）
- **核心能力**：多声道混音、实时 Gain 节点交叉淡入淡出、BiquadFilterNode 动态滤波、全景声 Panning
- **全局状态变量**：`chaosLevel` (0-100，映射游戏时间 0-10 分钟)

## 二、场景音频设计

### 2.1 卧室 (Bedroom) - 开场场景

#### 环境底噪
| 阶段 | 音频内容 | 技术实现 |
|------|----------|----------|
| 开场苏醒 | 沉重缓慢的钟摆声 "嗒、嗒、嗒" | 加载 `ambient_clock.mp3` 循环播放，音量 0.6 |
| 混沌阶段二 | 钟摆声逐渐失真、变慢 | BiquadFilterNode lowpass 频率从 2000→300 Hz，playbackRate 从 1.0→0.6 |
| 混沌阶段三 | 钟摆声扭曲、额外回声叠加 | DelayNode (delayTime: 0.5s, feedback: 0.4) + 高音噪声层 |

| 事件 | 音效 | 触发方式 |
|------|------|----------|
| 点击热区箭头 | `click_tick.mp3` 轻击声 | AudioManager.play('click_tick') |
| 蜡笔猜对 → 回忆触发 | `recall_crayon.mp3` 淡入渐强 | AudioManager.play('recall_crayon', 0.3) |

## 八、注意事项

1. **浏览器自动播放策略**：AudioContext 必须在用户首次交互（click/touch）后创建。在 `startNewGame()` 或 `btnNewGame` 第一次点击时执行 `AudioManager.init()`。
2. **资源预加载**：所有 MP3 文件在 `AudioManager.init()` 中使用 `fetch()` + `AudioContext.decodeAudioData()` 一次性解码为 AudioBuffer 放入缓存 Map。
3. **性能优化**：程序化生成音频节点用完即 `disconnect()` + `stop()`；长时间运行的 OscillatorNode 每 30 秒重置一次防内存泄漏。
4. **无障碍**：提供全局静音按钮（已有 `S.sound` + `btnSound` 逻辑），静音时调用 `AudioContext.suspend()`。
5. **回退方案**：如果 `AudioContext` 创建失败或 MP3 加载失败，静默 catch 错误，游戏照常运行不报错。
