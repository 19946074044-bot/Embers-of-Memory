# 记忆余烬 - 结局流程文档 (Endings Flowchart)

## 1. 结局前置核心机制：镜子双重校验 (The Mirror Decision)

| 项目 | 内容 |
| --- | --- |
| **发生场景** | 场景三：洗手间 (Washroom) |
| **交互物件** | 梳妆镜 (`./assets/items/washroom/vanity_mirror.png`) |
| **全局状态变量** | `isTrueEndRoute` (Boolean, 初始为 `false`) |
| **依赖计数器** | `collectedMemoriesCount` (已触发回忆的统计总数，满额为 5) |

### 镜子前身份提交判定分支

```text
【洗手间镜子放大层】
    │
    玩家在文本框中输入姓名（假设输入变量为 inputName）并提交
    ↓
    ├── 【分支 A：真结局激活 —— 双重强校验完全通过】
    │     ├── 触发条件：collectedMemoriesCount == 5 && inputName.toLowerCase() == "chenhuijun"
    │     ├── 叙事反馈：底部对话框跳出最清晰、坚定的手写体内心独白：“对，我是陈慧君。”
    │     └── 核心全局变量更新：isTrueEndRoute = true
    │
    └── 【分支 B：坏结局分流 —— 任意条件未达成（失败判定）】
          ├── 触发条件：collectedMemoriesCount < 5 || inputName.toLowerCase() != "chenhuijun"
          ├── 叙事反馈：不显示清醒对白，直接跳出主人公身份认知进一步解构的痛苦失语闪烁。
          └── 核心全局变量更新：isTrueEndRoute = false

```

#### ➔ 终极判定提交后的【失语文本吞噬演出与独立返回规则】：

1. **失语吞噬演出**：姓名变量录入完毕后，镜子界面上的所有文本和输入框瞬间开始疯狂错位、解构。系统利用 `innerHTML` 机制在屏幕最前端疯狂、死循环地克隆叠加乱码独白：“我是谁……谁在叫我……好难受……小雪……门开了吗……” 整个屏幕在 3 秒内被密密麻麻、重叠在一起的黑白文本粒子完全吞噬。
2. **交互终止（核心逻辑）**：吞噬演出结束后，遮罩层重置关闭。**不论玩家刚才输入了什么，该镜子位置的【黑色交互热区箭头永久消失】！**
3. **空间自跳转（去传送机制）**：**系统绝对不进行任何强制场景切换和场景重定向。** 玩家必须在交互结束后，手动点击洗手间左侧的白色导航箭头（`arrow_left.png`），自己跳转退回到【场景二：客厅】。

## 2. 终极通道：客厅出口大门 (The Exit Door)

* **激活时机**：玩家从洗手间自行跳转退回客厅后。
* **热区位置**：客厅画布正前方/出口大门边缘（基于 16:9 画布绝对定位）。
* **热区箭头图片**：`./assets/ui/arrow_white_exit.png`
* **触发机制**：此时洗手间已被玩家探索完毕且镜子热区失效。**玩家必须亲手点击客厅正前方这扇大门的白色通道箭头**，系统才会即时读取 `isTrueEndRoute` 的布尔值，切入对应的最终结局演出画面。

## 3. 结局演出详细流程

### 结局一：真结局 ——「记忆余烬」 (True End)

* **触发条件**：点击客厅大门时，满足双重校验（`isTrueEndRoute == true`）。

#### 🎬 演出分解步骤：

1. **场景重定向**：点击客厅大门白色箭头后，网页画布平滑淡出，即时载入真结局专属大门打开底图。
* 📷 **场景图片资产**：`./assets/scenes/endings/door_open_true.png` （一扇向外推开、门外透出温暖微光的70年代老式木门场景照片）。


2. **光效粒子渲染**：通过 CSS/Canvas 动画，从图片中“门”的坐标中心点开始，向外发散出柔和的**白色扩散光效（White Flare Effect）**。光效在 3 秒内逐渐扩大，最终将整个 16:9 网页画面完全吞噬、渐变为**纯白色**。
3. **叙事对白涌现**：画面变为纯白后，屏幕中央载入 UI 对话框（`dialog_bg.png`），以流式手写体风格输出主人公与女儿小雪的历史重逢原音对白：
* 💬 **内心对白**：
> “妈妈，你怎么一个人走到这里来啦？手怎么这么冰……”
> “小雪……妈妈今天，好像把你的名字记住了……”




4. **自动退出/结束**：对白播放完毕 **5 秒后**，画面平滑淡入黑底滚动演职员表（Credits），游戏自动退出至主菜单。

---

### 结局二：坏结局 ——「门外的色彩」 (Bad End)

* **触发条件**：点击客厅大门时，未能满足双重校验中的任意一项（`isTrueEndRoute == false`）。

#### 🎬 演出分解步骤：

```text
【点击客厅大门白色箭头（Bad End 触发）】
    │
    ▼
【第一阶段：大门开启】
    │   • 网页画布剧烈震颤（Screen Shake）并平滑淡出。
    │   • 载入大门打开的冰冷写实特写底图。
    │   • 📷 场景图片资产：`./assets/scenes/endings/door_open_bad.png`（推开后通往冰冷漆黑楼道的特写照片）。
    │
    ▼
【第二阶段：认知解构与双图闪回（核心高潮）】
    │   • 触发机制：系统通过 JS 定时器（setInterval）启动高频交替动画。
    │   • 📷 核心闪回资产 A：`./assets/scenes/endings/bad_chroma_distorted.png`（彩色高饱和度扭曲背景）
    │   • 📷 核心闪回资产 B：`./assets/scenes/endings/bad_mono_distorted.png`  (黑白高对比度扭曲背景)
    │   • 🎬 视觉特效：两张背景图在【100ms - 250ms】之间进行无序随机的高频交替切换闪回。
    │     配合 CSS filter 产生的画面数字撕裂卡顿（Glitch Effect），营造出极端痛苦、精神撕裂的视觉氛围。
    │   • 💬 底部对话框（dialog_bg.png）手写体文字高频闪烁，输出主角以为获得自由的错觉：
    │         “门开了……我终于逃出来了。外面好热闹，到处都是颜色……”
    │         “我自由了。我是晓雪，我要去上学了……”
    │   • ⏳ 持续时间：此高频交替闪回效果与第一阶段独白持续播放 6 秒。
    │
    ▼
【第三阶段：残酷现实与标题定格】
        • 定时器关闭（clearInterval），扭曲闪回骤停！
        • 整个网页画面瞬间失去所有画面色彩，不再展示任何写实切片，强行抹除背景，转为纯黑画布（#000000）。
        • 📷 最终定格资产：`./assets/scenes/endings/bad_title_screen.png`
          （纯黑底图正中央渲染出带有粗糙颗粒感、如同灰烬消散的白色美术大字标题 ——《门外的色彩》）。
        • 💬 底部对话框样式改变，转换为如同报纸新闻播报的冰冷宋体字（拒绝感性的文字风格）：
              “【寻人启事】陈慧君，女，78岁，患有严重阿尔茨海默症。于今日下午从家中走失，走失时身穿灰色针织衫……如有知情者，请速与家人联系。她的女儿一直在等她回家。”
        • ⏳ 自动退出机制：该结局标题与新闻启事在屏幕常驻显示 5 秒钟 后，底部对话框消隐，画面彻底淡出入终局，游戏通关结束。

```
