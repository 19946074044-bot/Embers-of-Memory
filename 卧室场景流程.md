#### B02 - 蜡笔 (Crayon)

| 属性 | 内容 |
| --- | --- |
| 热区位置 | 床头柜左侧（基于 16:9 画布的绝对定位百分比） |
| 美术资源（锁定态） | `./assets/items/bedroom/crayon_blurred.png`（带有黑白刮痕） |
| 交互机制分类 | 数字语义溶解系统（Interactive Typography） |

**交互流程（自适应全局混沌度）：**

```text
【第一层】网页场景视图 (index.html 主画布)
    │
    玩家点击蜡笔热区
    ↓
【第二层】物品放大悬浮窗 (<div id="overlay">)
    │
    ├── 渲染黑白手绘放大图：crayon_blurred.png
    └── 动态加载交互文本区域 (<div id="interaction-text">)
            │
            ├── IF chaosLevel <= 30 (阶段一：秩序)
            │     ├── 显示清晰选项：[A. 蜡笔] [B. 口红] [C. 铅笔]
            │     └── 玩家点击 [A. 蜡笔] ➔ 顺利播放回忆动画
            │
            ├── IF chaosLevel 31-70 (阶段二：瓦解)
            │     ├── JS 脚本启动：将选项拆解为 <span> 字符
            │     ├── 玩家鼠标靠近 [A. 蜡笔] 时，文字受 CSS transform 驱动向外弹开 50px
            │     ├── 文字内容产生语境混淆：选项字面随机在“蜡笔”与“女儿”之间闪烁
            │     └── 玩家必须快速且精准地“捕捉”点击 ➔ 触发带有画面撕裂效果的回忆
            │
            └── IF chaosLevel > 70 (阶段三：涌现)
                  ├── 选项文字完全脱离按钮框，在整个网页屏幕上做无序布朗运动
                  ├── 无论玩家点击哪里，交互区域均通过 innerHTML 疯狂堆叠输出内心独白：
                  │   "画画...晓雪...红色的...找不到笔了...我是谁..."
                  └── 文本在网页上形成视觉重叠（Text Overlapping），直到遮罩层自动崩溃

📷 **回忆图片位置：** `./assets/memories/memory1_crayon.png`

#### 交互流程 B：解锁态（持有钥匙并成功应用后）

* **解锁后特写照片资产**：`./assets/items/bedroom/nightstand_opened.png`（抽屉拉开，里面静静躺着一枚婚戒和一把挂着标签的老钥匙特写）
* **模糊戒指选项图片资产**：`./assets/items/bedroom/ring_blurred.png`（针对抽屉内老钥匙的模糊、带有划痕的局部特写，供认物解谜使用）
* **清晰戒指选项图片资产**：`./assets/items/bedroom/ring.png`（针对解谜完成后物品显现）

* **触发式回忆资产路径**：`./assets/memories/memory5_wedding.png`（主人公年轻时婚礼上的黑白纪实切片照片）

```text
【第一层】卧室场景视图
    │
    玩家将右侧工具栏的【床头柜钥匙图标】拖拽/点击应用至床头柜黑色热区
    ↓
【第二层】物品放大悬浮窗开启 (<div id="overlay">)
    │
    ├── 1. 底部对话框跳出内心独白：“里面有一枚戒指和一把钥匙……”
    │      └── 玩家点击任意空白处推进独白
    ↓
【语义溶解认物界面载入】
    │
    ├── 悬浮窗内即时切换渲染【模糊戒指图片】：./assets/items/bedroom/key_blurred.png
    └── 动态加载受时间混沌值控制的文本区域 (<div id="interaction-text">)
            │
            ├── IF chaosLevel <= 30 (阶段一：秩序)
            │     ├── 显示清晰的手写体选项：[A. 戒指] [B. 手镯] [C. 硬币]
            │     └── 玩家点击正确选项 [A. 戒指] ➔ 选项成功点击！
            │
            ├── IF chaosLevel 31-70 (阶段二：瓦解)
            │     ├── 文字选项拆解为 <span> 字符粒子的物理错位态
            │     ├── 玩家鼠标靠近 [A] 时，文字受算法驱动在“戒指”与“痛苦”之间高频闪烁并弹开
            │     └── 玩家必须精准捕捉并完成点击 [A. 戒指] ➔ 选项成功点击！
            │
            └── IF chaosLevel > 70 (阶段三：终极涌现)
                  ├── 【视觉干扰】：文字完全脱离按钮框在全屏做密集的无序布朗运动，利用 innerHTML 疯狂高频重叠堆叠输出：
                  │   “我的头好痛，抗拒着回忆起来……”
                  ├── 【底层交互】：正确选项 [A. 戒指] 按钮的 click 事件监听器保持激活，在屏幕上无序飘散。
                  └── 玩家顶着巨大视觉干扰，在乱码中精准“捕捉”并成功点击到移动中的正确选项 [A] ➔ 选项成功点击！

```
